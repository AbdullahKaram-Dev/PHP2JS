Introducción Ùltima Actualización: 2023-09-30

Si bien hoy en día existen múltiples frameworks para el frontend, entendemos que los monolitos creados con Laravel Framework continuarán existiendo durante mucho tiempo, y esto no es en absoluto negativo. La solución más efectiva no siempre radica en utilizar un framework de última generación. En ocasiones, resulta más eficiente para un equipo de desarrollo aprovechar las tecnologías estándar y el conocimiento que ya poseen.

Empleando tecnologías como jQuery, JavaScript Vanilla, HTML, CSS y el motor de vistas Blade de Laravel, se pueden lograr resultados sorprendentes y altamente funcionales. Sin embargo, sentíamos que faltaba algo: una forma sencilla, elegante y eficiente de compartir datos desde el servidor con el frontend desarrollado en JavaScript. No nos parecía cómodo insertar fragmentos de JavaScript en nuestras vistas, siempre buscábamos separar esta lógica en archivos independientes.

Fue entonces cuando comenzaron a aumentar las solicitudes al servidor para obtener variables que ya estaban presentes en la vista. Después de mucho tiempo reflexionando sobre cuál era la solución más efectiva, decidimos crear esta solución que estandariza y simplifica el proceso de compartir datos desde PHP con JavaScript.

Es posible que te preguntes cómo es factible hacerlo dado que son enfoques diferentes en cada lenguaje. Permítenos decirte que también enfrentamos ese desafío, pero ahora lo hemos resuelto.

Disfruta de la facilidad de crear una aplicación web sin sobrecargar el servidor con solicitudes innecesarias.

Requerimientos

Ahora que desees emplear esta solucion debes tener presente cumplir con lo siguiente:

PHP

Versión igual o superior a 7.4 y 8.0

Laravel Framework

Versión igual o superior a 9.0

Instalación

Este paquete esta disponible via Composer podras instalar la ultima version estable del paquete ejecutando el siguiente comando en tú proyecto de Laravel Framework:

composer require rmunate/php2js

Asegúrate de que en el composer.json tengas la biblioteca en la última versión. "rmunate/php2js": "^3.8".

Siempre luego de instalar o actualizar ejecuta el comando. 

php artisan php2js:clear

Versiones

Durante el proceso de creación de esta solución, se han lanzado varias versiones de la herramienta. Aunque las versiones anteriores continúan siendo compatibles con las últimas actualizaciones, es importante destacar que esta documentación se centra en las directivas y métodos disponibles en la versión actual. Se recomienda utilizar exclusivamente las directivas documentadas en esta guía, ya que los métodos y directivas anteriores pueden no estar documentados ni ser tan efectivos.

Retornar Vista

En versiones anteriores, solíamos utilizar la clase Rmunate\Php2Js\Render;. Esta clase ofrecía diversas soluciones mediante sus propios métodos fluidos para crear respuestas al frontend, facilitando la entrega de variables a JavaScript. Aunque este enfoque sigue siendo funcional y puede resultar útil en situaciones en las que ya se ha implementado o en las que se desee utilizar la clase que seguirá estando disponible, ahora ofrecemos una solución más sencilla.

En lugar de eso, aprovechamos el retorno bajo la clase Illuminate\View\View nativa del framework Laravel, lo que simplifica significativamente el proceso.

El siguiente código muestra cómo retornar una vista de forma convencional, sin compartir datos con JS, manera nativa del marco, es fundamental que se entiendan las diversas maneras de usarse, ya que dependemos de esto para poder pasar a explicar como se comparten los valores con JavaScript.

1. Usando el método "view" con "compact"

class YourController extends Controller
{
	public function index()
	{
		$nombre = 'PHP2JS';
		return view('welcome', compact('nombre'));
	}
}
2. Usando solo el método "view" sin "compact"

class YourController extends Controller
{
	public function index()
	{
		$valor = 'PHP2JS';
		return view('welcome', [
			'nombre' => $valor
		]);
	}
}
3. Usando el método "view", el método "with" y "compact"

class YourController extends Controller
{
	public function index()
	{
		$nombre = 'PHP2JS';
		return view('welcome')->with(compact('nombre'));
	}
}
4. Usando el método "view", el método "with" sin "compact"

class YourController extends Controller
{
	public function index()
	{
		$valor = 'PHP2JS';
		return view('welcome')->with([
			'nombre' => $valor
		]);
	}
}

Compartir Valores Con JavaScript

Entendiendo lo anterior, ahora pasaremos a exponer como podras usar culquiera de las cuatro anteriores opciones para compartir las variables que se retornan a la vista con JavaScript.

Al instalar este paquete, ahora tendrás disponible el método ->toJS(). Este método permite que todas las variables que se hayan pasado a la vista se compartan directamente con los scripts escritos tanto en el mismo archivo de vista Blade como en archivos externos de JavaScript que se agreguen mediante la sintaxis:
<script src="{{ asset('miarchivo.js') }}"></script>

Veamos un ejemplo: 


class YourController extends Controller
{
	public function index()
	{
		return view('welcome')->with([
			'nombre' => 'Raul Uñate'
		])->toJs();
	}
}

Recuerda que puedes usar cualquiera de las cuatro formas de pasar variables a la vista explicadas en Retorno De Vista

Lo anterior genera que inmediatamente en tus scripts de JavaScript, puedas leer las variables retornadas del controlador a traves de una constante con el nombre PHP2JS 


let nombre = PHP2JS.vars.nombre;
// 'Raul Uñate'

Tip De Seguridad

Puedes emplear una muy buena practica y es asignarle un nombre diferente a la constante que se recibirá en el contexto de JavaScript, esto con el fin de que solo los desarrolladores o mantenedores, conozcan el medio de entrada a los valores retornados por el backend.

Asignar Un Alias Unico En JavaScript

Por defecto, el paquete permite que las variables devueltas desde el backend sean accesibles en el contexto de JavaScript a través de la constante PHP2JS. Sin embargo, tienes la flexibilidad de asignarle un nombre diferente, uno que sea más descriptivo o acorde a su uso. Por ejemplo:

  • __PHP
  • __Laravel
  • __Back
  • __MiAlias
¿Cómo Asignarle Un Nombre Personalizado?

Simplemente, como argumento del método ->toJs(), proporcionarás el nombre de la constante que deseas utilizar. Ten en cuenta que este nombre estará reservado a partir de este punto y no podrás volver a declarar la constante en tus archivos de JavaScript. 


class YourController extends Controller
{
	public function index()
	{
		return view('welcome')->with([
			'nombre' => 'Raul Uñate'
		])->toJs("__PHP");
	}
}

Ahora podras acceder a los valores a traves de esta constante en JavaScript: 


let nombre = __PHP.vars.nombre;
// 'Raul Uñate'
