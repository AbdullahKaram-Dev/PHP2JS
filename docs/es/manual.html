Introducción Ùltima Actualización: 2023-09-30

Si bien hoy en día existen múltiples frameworks para el frontend, entendemos que los monolitos creados con Laravel Framework continuarán existiendo durante mucho tiempo, y esto no es en absoluto negativo. La solución más efectiva no siempre radica en utilizar un framework de última generación. En ocasiones, resulta más eficiente para un equipo de desarrollo aprovechar las tecnologías estándar y el conocimiento que ya poseen.

Empleando tecnologías como jQuery, JavaScript Vanilla, HTML, CSS y el motor de vistas Blade de Laravel, se pueden lograr resultados sorprendentes y altamente funcionales. Sin embargo, sentíamos que faltaba algo: una forma sencilla, elegante y eficiente de compartir datos desde el servidor con el frontend desarrollado en JavaScript. No nos parecía cómodo insertar fragmentos de JavaScript en nuestras vistas, siempre buscábamos separar esta lógica en archivos independientes.

Fue entonces cuando comenzaron a aumentar las solicitudes al servidor para obtener variables que ya estaban presentes en la vista. Después de mucho tiempo reflexionando sobre cuál era la solución más efectiva, decidimos crear esta solución que estandariza y simplifica el proceso de compartir datos desde PHP con JavaScript.

Es posible que te preguntes cómo es factible hacerlo dado que son enfoques diferentes en cada lenguaje. Permítenos decirte que también enfrentamos ese desafío, pero ahora lo hemos resuelto.

Disfruta de la facilidad de crear una aplicación web sin sobrecargar el servidor con solicitudes innecesarias.

Requerimientos

Ahora que desees emplear esta solucion debes tener presente cumplir con lo siguiente:

PHP

Versión igual o superior a 7.4 y 8.0

Laravel Framework

Versión igual o superior a 9.0

Instalación

Este paquete esta disponible via Composer podras instalar la ultima version estable del paquete ejecutando el siguiente comando en tú proyecto de Laravel Framework: 

composer require rmunate/php2js

Asegúrate de que en el composer.json tengas la biblioteca en la última versión. "rmunate/php2js": "^3.8".

Siempre luego de instalar o actualizar ejecuta el comando. 

php artisan php2js:clear

Retornar Vista

En versiones anteriores, solíamos utilizar la clase Rmunate\Php2Js\Render;. Esta clase ofrecía diversas soluciones mediante sus propios métodos fluidos para crear respuestas al frontend, facilitando la entrega de variables a JavaScript. Aunque este enfoque sigue siendo funcional y puede resultar útil en situaciones en las que ya se ha implementado o en las que se desee utilizar la clase que seguirá estando disponible, ahora ofrecemos una solución más sencilla.

En lugar de eso, aprovechamos el retorno bajo la clase Illuminate\View\View nativa del framework Laravel, lo que simplifica significativamente el proceso.

El siguiente código muestra cómo retornar una vista de forma convencional, sin compartir datos con JS, manera nativa del marco, es fundamental que se entiendan las diversas maneras de usarse, ya que dependemos de esto para poder pasar a explicar como se comparten los valores con JavaScript.

1. Usando el método "view" con "compact"

class YourController extends Controller
{
	public function index()
	{
		$nombre = 'PHP2JS';
		return view('welcome', compact('nombre'));
	}
}
2. Usando solo el método "view" sin "compact"

class YourController extends Controller
{
	public function index()
	{
		$valor = 'PHP2JS';
		return view('welcome', [
			'nombre' => $valor
		]);
	}
}
3. Usando el método "view", el método "with" y "compact"

class YourController extends Controller
{
	public function index()
	{
		$nombre = 'PHP2JS';
		return view('welcome')->with(compact('nombre'));
	}
}
4. Usando el método "view", el método "with" sin "compact"

class YourController extends Controller
{
	public function index()
	{
		$valor = 'PHP2JS';
		return view('welcome')->with([
			'nombre' => $valor
		]);
	}
}

Compartir Valores Con JavaScript

Entendiendo lo anterior, ahora pasaremos a exponer como podras usar culquiera de las cuatro anteriores opciones para compartir las variables que se retornan a la vista con JavaScript.

Al instalar este paquete, ahora tendrás disponible el método ->toJS(). Este método permite que todas las variables que se hayan pasado a la vista se compartan directamente con los scripts escritos tanto en el mismo archivo de vista Blade como en archivos externos de JavaScript que se agreguen mediante la sintaxis:
<script src="{{ asset('miarchivo.js') }}"></script>

Veamos un ejemplo: 


class YourController extends Controller
{
	public function index()
	{
		return view('welcome')->with([
			'nombre' => 'Raul Uñate'
		])->toJs();
	}
}

Recuerda que puedes usar cualquiera de las cuatro formas de pasar variables a la vista explicadas en Retorno De Vista

Lo anterior genera que inmediatamente en tus scripts de JavaScript, puedas leer las variables retornadas del controlador a traves de una constante con el nombre PHP2JS 


let nombre = PHP2JS.vars.nombre;
// 'Raul Uñate'

Tip De Seguridad

Puedes emplear una muy buena practica y es asignarle un nombre diferente a la constante que se recibirá en el contexto de JavaScript, esto con el fin de que solo los desarrolladores o mantenedores, conozcan el medio de entrada a los valores retornados por el backend.

Asignar Un Alias Unico En JavaScript

Por defecto, el paquete permite que las variables devueltas desde el backend sean accesibles en el contexto de JavaScript a través de la constante PHP2JS. Sin embargo, tienes la flexibilidad de asignarle un nombre diferente, uno que sea más descriptivo o acorde a su uso. Por ejemplo:

  • __PHP
  • __Laravel
  • __Back
  • __MiAlias
¿Cómo Asignarle Un Nombre Personalizado?

Simplemente, como argumento del método ->toJs(), proporcionarás el nombre de la constante que deseas utilizar. Ten en cuenta que este nombre estará reservado a partir de este punto y no podrás volver a declarar la constante en tus archivos de JavaScript. 


class YourController extends Controller
{
	public function index()
	{
		return view('welcome')->with([
			'nombre' => 'Raul Uñate'
		])->toJs("__PHP");
	}
}

Ahora podras acceder a los valores a traves de esta constante en JavaScript: 


let nombre = __PHP.vars.nombre;
// 'Raul Uñate'

Compartir Solo Algunos Valores Con JavaScript

Si lo que requieres es no compartir el total de las variables retornadas a la vista con JavaScript, sino en su lugar solo un grupo de valores específicos, entonces te será más conveniente usar el método ->toStrictJS(). Este método funciona de una manera similar al método ->with() propio de Laravel Framework.

Veamos un ejemplo donde a la vista y a JavaScript se les comparten valores diferentes. 


class YourController extends Controller
{
	public function index()
	{
		//Recuerda que también puedes usar compact si ya tienes definidas las variables.
		return view('welcome')->with([
			'paquete' => 'PHP2JS',
			'version' => '^3.8',
		])->toStrictJS([
			'link' => 'https://github.com/rmunate/PHP2JS'
		]);
	}
}

Con este ejemplo solo estamos compartiendo a JavaScript el valor del Link.

Tip De Seguridad

Puedes asignarle un nombre diferente a la constante que se recibirá en el contexto de JavaScript, simplemente pásale como segundo argumento al método el nombre a usar. ->toStrictJS([...], "__PHP")

Bloques De Información Preconstruidos

PHP2JS trae de manera preconstruida algunos bloques de datos que consideramos podrán darte un valor agregado en ciertos tipos de desarrollo. Hemos dispuesto ciertos bloques que te mostraremos en este apartado, los cuales podrás atar al retorno de la vista para compartirlos con JavaScript. El método ->attach() nos presta la funcionalidad de agregar estos datos al retorno de la Vista.

El método lo podrás usar después de los métodos ->toJS() ó ->toStrictJS(), como se muestra en el ejemplo: 


class YourController extends Controller
{
	public function index()
	{
		return view('welcome')->with([
			'nombre' => 'Raul Uñate'
		])->toJs("__PHP")->attach('agent', 'user', 'csrf', ...);
	}
}

Veamos los valores actualmente disponibles, recuerda que puedes aportar en el paqute, ajustando los valores actualmente disponibles o sugiriendo nuevos.

agent Podrás conocer desde qué dispositivo se ejecuta la conexión, el navegador en uso, si es una conexión desde un dispositivo móvil, la dirección IP de origen de la conexión y el puerto. 

{
	identifier:  // Agente Registrado Desde El Servidor,
	remote_ip:   // IP Remota de conexión del usuario,
	remote_port: // Puerto Remoto de conexión del usuario,
	browser:     // Datos del Navegador en uso (Nombre, Versión y Plataforma),
	isMobile:    // Valida se la conexión es desde un dispositivo móvil.
	OS:          // Sistema operativo empleado para conectarse a la aplicación.
}
url Podrás conocer los datos de la URL en uso, los parámetros pasados, la URI, la base URL para peticiones al servidor, etc. 

{
	baseUrl:      // Base URL que se empleará para peticiones AJAX, Axios, Fetch, etc.,
	fullUrl:      // URL Completa con argumentos por Query,
	uri:          // URI, valor que está después del nombre del Dominio,
	parameters: {
		route:    // Parámetros pasados por la ruta de Laravel (GET),
		get:      // Parámetros enviados por URL (Query) (GET),
		post:     // Parámetros enviados por (POST),
	},
	scheme:       // Esquema HTTP en uso.,
	currentName:  // Nombre de la Ruta Asignado Desde Las Rutas De Laravel,
	isSecure:     // Define si cuenta con SSL,
}
csrf Retorna un token válido para peticiones desde JavaScript, lo que permite hacer peticiones Ajax, Axios, Fetch agregando el token dentro de la carga útil. 

{
	token:          // Token válido para peticiones al servidor por Request de Laravel,
	tokenCookie:    // Token entregado por Cookie desde el servidor,
}
framework Retorna información referente al framework en uso, como la versión y los valores no sensibles del ENV. 

{
	version:        // Versión de Laravel en Uso
	environment: {
		name:       // Nombre de la aplicación desde el ENV
		debug:      // Estado del Debugger de la aplicación.
		context:    // Ambiente en el que se ejecuta de acuerdo al ENV
		url:        // URL de la aplicación puesta en el ENV
	}
}
php Retorna información referente a la versión de PHP en uso y detalles del servidor. 

{
	id:                     // Versión ID de PHP
	version:                // Versión en uso de PHP
	release:                // Release Actual de PHP
	serverSoftware:         // Software del servidor web que está ejecutando el script actual
	serverOperatingSystem:  // Sistema Operativo Sobre el Cual Corre PHP
	extensions:             // Extensiones habilitadas de PHP
	clientLanguage:         // Lenguaje Habilitado del Cliente
}
user Retorna la información del usuario en sesión. El ID se retorna encriptado para evitar mostrar información sensible, y no se incluyen datos como contraseña ni marcas de tiempo. 

{
	id:                     // Id Encriptado
	name:                   // Nombre Usuario
	email:                  // Correo electrónico
	...                     // Todos los demás datos no sensibles.
}

¿Cómo Accederlos?

Bajo el mismo alias que hayas usado en el metodo ->toJs("__PHP") o en el metodo ->toStrictJS([...], "__PHP") podras accederlos desde JavaScript. Por ejemplo quisieras acceder a la información del usuario podrias hacerlo así: __PHP.user

Compartir Variables Existentes en PHP con JavaScript

La directiva principal de la Biblioteca @PHP2JS_VARS(). Te permite compartir con JavaScript todas las variables existentes en PHP. Esto significa que le pasarás a JavaScript los valores retornados por el controlador, así como los valores de las variables que hayas creado en la vista Blade.

¡Así es! Si has definido bucles y demás sentencias en el Front, todo esto lo tendrás a la mano en JavaScript. Esto es supremamente útil. 


// Alias por defecto PHP2JS
@PHP2JS_VARS() 

// Con Alias Personalizado
@PHP2JS_VARS('__PHP')

¡Recuerda!

No puedes emplear alias que coincidan con constantes ya definidas en el contexto de JavaScript, por lo cual si esta misma vista la retornaste compartiendo datos con JavaScript, recuerda usar un alias diferente.

Compartir Solo Algunas Variables Existentes en PHP con JavaScript.

La segunda directiva principal de la Biblioteca @PHP2JS_VARS_STRICT(['variable1','variable2']). Te permite compartir con JavaScript solo algunas variables existentes en PHP. Esto significa que le pasarás a JavaScript los valores controlados que requieras.

Esto es más que útil, seguro, podrás pasar solo algunos datos, evitando pasar al contexto de JavaScript más valores de los necesarios. 


// Alias por defecto PHP2JS
@PHP2JS_VARS_STRICT(['variable1','variable2'])

// Con Alias Personalizado
@PHP2JS_VARS_STRICT(['variable1','variable2'], '__PHP')

¡Recuerda!

No puedes emplear alias que coincidan con constantes ya definidas en el contexto de JavaScript, por lo cual si esta misma vista la retornaste compartiendo datos con JavaScript, recuerda usar un alias diferente.

Compartir Los Datos Del Agente De Conexión

Esta directiva te permite pasar un objeto con los datos del agente de conexión capturados por el servidor a JavaScript. No se emplea ningún método de captura con JS, todos los datos son leídos y pasados por el servidor a JavaScript. 


// Alias por defecto PHP2JS
@PHP2JS_AGENT() 

// Con Alias Personalizado
@PHP2JS_AGENT('__PHP')

¡Recuerda!

No puedes emplear alias que coincidan con constantes ya definidas en el contexto de JavaScript, por lo cual si esta misma vista la retornaste compartiendo datos con JavaScript, recuerda usar un alias diferente.

Compartir Los Datos De La URL

Esta directiva te permite pasar un objeto con los datos de la URL en uso a JavaScript. El objeto te permitirá acceder a los valores similar a cuando envías una petición al servidor. 


// Alias por defecto PHP2JS
@PHP2JS_URL() 

// Con Alias Personalizado
@PHP2JS_URL('__PHP')

¡Recuerda!

No puedes emplear alias que coincidan con constantes ya definidas en el contexto de JavaScript, por lo cual si esta misma vista la retornaste compartiendo datos con JavaScript, recuerda usar un alias diferente.

Compartir Token Valido

Esta directiva es de las más usadas, permite contar con un token de Laravel válido desde el contexto de JavaScript. El objeto contendrá un token válido al igual que el token entregado por cookie desde el Servidor. 


// Alias por defecto PHP2JS
@PHP2JS_CSRF() 

// Con Alias Personalizado
@PHP2JS_CSRF('__PHP')

¡Recuerda!

No puedes emplear alias que coincidan con constantes ya definidas en el contexto de JavaScript, por lo cual si esta misma vista la retornaste compartiendo datos con JavaScript, recuerda usar un alias diferente.

Compartir Información Del Framework

Esta directiva permite conocer algunos datos inherentes al Framework. Permite definir en muchos casos, según se ha medido su uso, para poder determinar si el debugger de la aplicación está activa para ejecutar algunas acciones desde JS. Puede que encuentres más utilidades. 


// Alias por defecto PHP2JS
@PHP2JS_FRAMEWORK() 

// Con Alias Personalizado
@PHP2JS_FRAMEWORK('__PHP')

¡Recuerda!

No puedes emplear alias que coincidan con constantes ya definidas en el contexto de JavaScript, por lo cual si esta misma vista la retornaste compartiendo datos con JavaScript, recuerda usar un alias diferente.

Compartir Información De PHP

Esta directiva permite conocer algunos datos inherentes a la versión de PHP en USO. Esta directiva puede ser empleada para sistemas donde se permite el uso de múltiples versiones de PHP. Solo úsala si realmente le tienes un uso. 


// Alias por defecto PHP2JS
@PHP2JS_PHP() 

// Con Alias Personalizado
@PHP2JS_PHP('__PHP')

¡Recuerda!

No puedes emplear alias que coincidan con constantes ya definidas en el contexto de JavaScript, por lo cual si esta misma vista la retornaste compartiendo datos con JavaScript, recuerda usar un alias diferente.

Compartir Información Del Usuario En Sesión

Esta directiva permite compartir los datos del usuario en sesión con JavaScript. Pensamos en dejarla por defecto, porque siempre los usuarios de la plataforma la emplean, sin embargo, preferimos que esté a tu discreción. Tendrás los datos del usuario en sesión en JavaScript, ¡calma!, el ID del usuario en la base de datos e información sensible no se mostrará. El ID lo tendrás encriptado por el método Crypt::encrypt() de Laravel. 


// Alias por defecto PHP2JS
@PHP2JS_USER() 

// Con Alias Personalizado

Limpiar

Limpiar Objeto

El método .clear() vaciará el objeto entregado por PHP a JavaScript. Esto será útil cuando desees que los valores no sean accesibles desde las diferentes referencias del objeto. 


// Vaciar El Objeto (Todas las referencias del Objeto perderán los valores)
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
PHP2JS.clear();

Limpiar Sin Funciones

El método .clearWithoutFunctions() vaciará las variables del objeto entregado por PHP a JavaScript. Pero no eliminará los métodos que se están listando en este apartado. Esto será útil cuando desees que los valores no sean accesibles desde las diferentes referencias del objeto. 


// Vaciar El Objeto (Las funciones seguirán existiendo)
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
PHP2JS.clearWithoutFunctions();

Asignar

Asignar Mismo Objeto

El método .assign() le asigna a una nueva variable o constante los mismos valores del Objeto entregado por PHP. Ahora podrás eliminar el objeto original sin perder los valores que dejas en el nuevo. Esto será útil cuando desees que los valores no sean accesibles desde las diferentes referencias del objeto original y en su lugar usar un objeto en tiempo de ejecución de JavaScript. 


// Asignar una copia del objeto a una nueva variable en tiempo de ejecución.
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
const __PHP = PHP2JS.assign();

// Aquí podrías vaciar el objeto original.
PHP2JS.clear();

// Los valores ahora estarán en "__PHP"

Asignar Y Limpiar

El método .assignAndClear() le asigna a una nueva variable o constante los mismos valores del Objeto entregado por PHP y adicional borra el elemento original. Esto será útil cuando desees que los valores no sean accesibles desde las diferentes referencias del objeto original y en su lugar usar un objeto en tiempo de ejecución de JavaScript. 


// Vaciar El Objeto (Todas las referencias del Objeto original perderán los valores)
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
const __PHP = PHP2JS.assignAndClear();

// Los valores ahora estarán en "__PHP"

Asignar Y Limpiar Sin Funciones

El método .assignAndClearWithoutFunctions() le asigna a una nueva variable o constante los mismos valores del Objeto entregado por PHP y adicional borra del elemento original los valores pero no las funciones o métodos. Esto será útil cuando desees que los valores no sean accesibles desde las diferentes referencias del objeto original y en su lugar usar un objeto en tiempo de ejecución de JavaScript. 


// Vaciar El Objeto (Sin eliminar las funciones o métodos.)
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
const __PHP = PHP2JS.assignAndClearWithoutFunctions();

// Los valores ahora estarán en "__PHP" y en "PHP2JS" seguirán existiendo los métodos del objeto.

Unicamente

Solo los valores definidos por el usuario

El método .only(...props) permite extraer la información del objeto entregado por PHP de los valores que se requieran. Como parámetros, podrás definir qué valores realmente necesitas. Esto será útil cuando no desees cargar todo el contenido de la constante. 


// Consulta solo los datos pasados como argumentos en el método Only
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
const post = PHP2JS.only('post');

Unicamente Funciones

El método .onlyFunctions() permite extraer la información de las funciones existentes en el objeto entregado por PHP a JavaScript. 


// Consulta solo los métodos o funciones del objeto entregado por PHP a JavaScript.
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
const functions = PHP2JS.onlyFunctions();

Excepto

Todo menos los valores definidos por el usuario

El método .except(...props) permite extraer la información del objeto entregado por PHP de los valores que no se requieran. Como parámetros, podrás definir qué valores no deseas que se tengan en cuenta. Esto será útil cuando no desees cargar todo el contenido de la constante. 


// Consulta solo los datos que requiera exceptuando otros.
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
const withoutNews = PHP2JS.except('news');

Excepto Funciones

El método .exceptFunctions() permite extraer la información de las variables y bloques de código preconstruidos exceptuando los métodos y funciones. 


// Consulta solo los valores del objeto entregado por PHP a JavaScript.
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
const values = PHP2JS.exceptFunctions();

Verificar Propiedad

Validar Si Existe Una Propiedad

El método .hasProperty('prop') permite saber si una propiedad existe dentro del objeto. El retorno será verdadero o falso según sea el caso. 


// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
const existPost = PHP2JS.hasProperty('post'); // true // false

Obtener

Obtener Todas Las Propiedades

El método .getAllProperties() getAllProperties 


// Crea un arreglo con todas las propiedades del objeto.
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
const existPost = PHP2JS.getAllProperties();

Obtener una propiedad especifica

El método .get('prop') permite obtener una propiedad en específico del objeto. Retorna solo la propiedad consultada. 


// En este ejemplo consultaremos solo el valor de "date"
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
PHP2JS.get("date");

Asignar Propiedad

Agregarle Una Nueva Propiedad

El método .set('prop','value') permite asignar un nuevo valor a una propiedad existente del objeto. Si deseas actualizar, reemplazar o asignar nuevos valores a una propiedad del objeto, podrás hacerlo fácilmente. 


// En este ejemplo agregaremos la propiedad "count" con el valor de 0;
// Recuerda que PHP2JS debe ser cambiado por el Alias que hayas usado.
PHP2JS.set("count", 0);
